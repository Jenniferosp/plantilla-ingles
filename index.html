Abrir contenido en ventana modal

Para abrir contenidos en ventanas modales dentro del mismo objeto. Se agrega la clase "w_content_controler" al elemento que debe abrir la ventana (origen), por defecto con el evento clic. No tiene que ser botón o enlace, puede ser cualquier elemento. Se debe establecer la propiedad "data-content" con la ruta CSS del elemento que se convertirá en ventana (destino). El elemento que se abre como ventana debe tener la clase "w_content". Se puede utilizar la propiedad "data-property-width", tanto en el origen como en el destino, para definir el ancho de la ventana modal y "data-property-height" para definir el alto. También se puede asignar una clase al destino mediante la propiedad "data-cssclass" en el elemento destino.

Ejemplo
<button class="w_content_controler general" data-content="#textos_modal_window">
    Abrir ventana modal
</button>
<div id="textos_modal_window" class="w_content">
    Así se ve la ventana modal
</div>

Diálogos

Para diálogos utilizar una tabla con la clase CSS dialogue. Cada línea del diálogo debe ir en una estructura de fila y la fila (tr) de uno de los personajes debe tener la clase odd, como lo siguiente: 

<tr class="odd">
    <th>Persona 1:</th>
    <td>Lo que dice</td>
</tr>
<tr>
    <th>Persona 2:</th>
    <td>Lo que dice la otra persona</td>
</tr>

Mouse "sobre" - Contenido siempre visible

Para desplegar contenido pasando el mouse por un elemento, se utiliza la clase "act_mouse_over_one" en un contenedor con elementos que adicionan la propiedad "data-ref", la cual debe contener la ruta CSS del elemento que se muestra al pasar el mouse. Se mantiene visible el último elemento mostrado.

Importante: No se controla la visualización inicial por lo que se deben ocultar los elementos explícitamente y mantener visible solamente el inicial. Además, marcar el elemento de control del elemento visible con la clase "current".

Audios

Para audios utilizar el siguiente botón: El atributo data-sound contiene la ruta compuesta por el nombre del archivo y si está dentro de una carpeta, el nombre de la carpeta.

Ejemplo
<button class="iconsound button" 
    data-sound="13/sonido1_slide_13">
</button>

Donde 13 es la carpeta y sonido1_slide_13 es el nombre del archivo sin la extensión. Todos los sonidos deben ir en content/sounds.

    4. Actividades

    Actividad de tipo cuestionario dentro del contenido

    Tipos de pregunta disponibles:

    • Etiqueta (label): No es calificable, se utiliza para mostrar un texto estático.
    • Selección única (simplechoice): Para seleccionar una opción correcta de una lista de posibles valores.
    • Selección múltiple (multichoice): Para seleccionar varias opciones, todas las correctas, en una lista de posibles valores.
    • Selección múltiple en conjunto (multisetchoice): Para seleccionar las correctas de una lista de posibles valores. Permite definir varios conjuntos correctos, excluyentes entre si.
    • Cloze (complete): Presenta varias opciones entre un contenido, con controles para seleccionar los valores correctos.
    • Definición (defineterm): Para permitir entradas de texto que tienen que coincidir completamente con las respuestas correctas, según el término a definir.

    Con el atributo data-intro definido a "false" se omite la pantalla de presentación y se muestran inmediatamente las preguntas. Es útil sobre todo cuando hay una sola pregunta.

    Ordenar

    Aplica para cualquier HTML y opera sobre los <li> que estén contenidos dentro de un <ul> con clase CSS "sortable". Los <li> deben tener un atributo data-position con el indice que indica la posición adecuada, la cual empieza desde cero.

    Si se requiere que el ordenamiento sea horizontal es preciso agregar al <ul> contenedor la clase sortable-horizontal. Se pueden tener varias listas horizontales o combinar horizontales con verticales, pero se debe mantener coherencia en el indice desde cero en la primera lista hasta la cantidad total de elementos ordenados en la última lista.

    5. Referencias

    5.1. Autoría de diseñador del proyecto

    Si la imagen es de autoría del diseñador (Vectores o ilustraciones) o hay una fotografía de autoría de uno de los integrantes del equipo de trabajo, se cita de la siguiente manera:

    Imagen #. Nombre de la imagen, (año).

    En el pie de imagen no se pone el autor, puesto que ya aparecerá en los créditos del OVA. Para fotografía, diagrama, gráfico o tabla se cita igual.

    Imagen 1. Marca, (2016).

    5.2. Autoría proveniente de la Web

    Si la imagen, fotografía, diagrama, gráfico o tabla proviene de la web se cita así:

    Imagen #. Autor, (año). Descripción de la imagen (Si la hay). Recuperado de: Nombre (enlace) del sitio de donde fue sacado y fecha de consulta.

    En caso de no tener autor se coloca descripción de la imagen, diagrama, fotografía, gráfico o tabla:

    Imagen #. Descripción de la imagen, (año). Recuperado de: Nombre (enlace) del sitio de donde fue sacado y fecha de consulta.
    Imagen #. Descripción de la imagen, (s.f) Recuperado de: Nombre (enlace) del sitio de donde fue sacado y fecha de consulta.

    Importante: se deberá hacer el hipervinculo al enlace que provee la descarga de la imagen desde Internet. En caso de no tener fecha, colocar (s.f)

    Imagen 1. HASHIGUCHI Mujer enpolvandose, (1921).
    Recuperado de:     Hashiguchi Goyo el 26 de abril de 2016.

    5.3. Autoría de libro

    Si la imagen, diagrama, gráfico o fotografía tiene derechos de licencia, se hace la cita del autor de la siguiente manera:

    Diagrama #. Autor, (año). Descripción de la imagen (si la hay).
    Diagrama #. Autor, (año).

    Importante: se deberá colocar al final Diseño adaptado en caso de que sea modificada un poco la imagen en cuanto a color y forma. En caso de no tener fecha, colocar (s.f)

    Diagrama 1. Schiffman, L. & Kanuk, L. (2010). Modelo del proceso de motivación. Diseño adaptado.
